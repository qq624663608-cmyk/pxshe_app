# pxshe 后台管理系统 使用文档

## 访问入口

| 域名 | 用途 |
|------|------|
| https://admin.pxshe.com | 后台管理 UI(admin-api:10009) |
| https://chat.pxshe.com | chat-api (10008,Flutter 普通用户) |
| https://api.pxshe.com | openim-api (10002,被 SDK 调) |
| **wss://ws.pxshe.com** | **openim-msggateway (10001,Flutter SDK 长连接)** |

> 完整服务清单见 [SERVICE_INVENTORY.md](SERVICE_INVENTORY.md)。

默认账号 `chatAdmin`,密码 `chatAdmin`。

显示 6 个业务数据统计 (宇宙/角色/故事/地点/武器/页面),每 30 秒自动刷新,可手动关闭自动刷新或立即刷新。

## 用户管理

- 搜索:输入 userID 或昵称,点击搜索图标或回车
- 添加用户:点击"添加用户" → 填 userID + 密码 + 昵称 + 头像 URL → 确定
- 编辑资料:点击行的"改资料"按钮 → 改昵称/头像 → 保存
- 重置密码:点击行的"重置密码"按钮 → 输入新密码 → 保存
- 封禁用户:点击行的"封禁"按钮 → 二次确认 → 用户被封禁 (不可登录)
- 详情:点击行的"详情"按钮 → 弹出 Modal 显示完整用户信息 JSON
- 导出 CSV:点击"导出 CSV" → 下载 users-yyyy-MM-dd.csv

---

## 在线状态

- 输入 userID (支持多个,逗号/换行/分号分隔),最多 200 个
- 点击"查询" → 表格显示每个用户的在线状态
  - `[在线]` Tag (绿) — 用户当前已连接 msg-gateway
  - `[离线]` Tag (灰) — 用户未连接
- 显示平台列:连接的平台列表 (iOS/Android/Web/PC)
- 显示最后在线时间
- 顶部统计:`共 N · 在线 M · 离线 K`
- 5 秒冷却:防止压垮 msg-gateway
- 导出 CSV:下载 online-status-yyyy-MM-dd.csv

## 通知账号 (Sender)

用于发送系统级广播的官方账号。3 个预置账号:

| userID | 用途 |
|--------|------|
| sys_notice | 系统公告/维护通知 |
| sys_security | 安全警告/密码过期 |
| sys_marketing | 运营通知/活动推广 |

- 列表:显示 userID、昵称、头像、级别 Tag
- 添加:点击"添加通知账号" → 填 userID + 昵称 + 头像 URL + 选级别 (默认 3)
- 编辑:点击行的编辑图标 → 改昵称/头像 → 保存
- 级别映射:1=应用管理员 / 3=通知管理员(默认) / 80=管理员 / 100=超级管理员

## 用户标签 (User Command)

用户云端 KV 存储,客户端 SDK 启动时拉取配置:

- 4 种类型:1=用户设置 (theme/language) / 2=游戏积分 (vip/score) / 3=业务运营 / 4=系统开关
- 添加:点击"添加标签" → 选 Type + 填 UUID + Value + 可选备注 → 保存
- 编辑:点击行的编辑图标 → 改 Value/备注 → 保存
- 删除:点击行的删除按钮 → 二次确认 → 删除
- 插入演示:点击"插入演示" → 一键为当前 userID 添加 5 条演示数据 (theme_color/language/push_enabled/vip_level/credit_score)

## 业务广播 (Broadcast)

用系统通知账号发送公告/营销消息:

1. 选择发送账号 (sys_notice / sys_security / sys_marketing)
2. 选择接收方类型:
   - 单个用户 — 输入 userID
   - 多个用户 — 输入多个 userID (逗号/换行分隔)
   - 多个群组 — 输入多个 groupID
   - 全平台 — 一键发送给所有用户
3. 选择消息类型:
   - 文本 (101) — 普通文本,直接输入内容
   - Markdown (118) — 富文本(部分客户端不渲染)
   - 自定义 (110) — JSON 结构,需客户端配合渲染
4. 高级选项:仅在线送达 / 不推送离线消息
5. 点击"立即发送" → 结果卡显示成功/失败数

## 通用操作说明

### 搜索
几乎所有列表页都支持搜索框,输入关键字后回车或点击搜索图标。

### 分页
表格底部分页器,可调整每页条数 (默认 20)。

### 刷新
每页右上角"刷新"按钮重新拉取数据。

### 删除二次确认
所有删除操作都要求输入关键字段确认 (如 userID、账号名),防止误删。

### 自动登录续期
登录后,如果距离 token 过期 < 5 分钟,会弹窗提醒输入账号密码续期; < 1 分钟未续期会强制跳登录页。

### 401 处理
如果 token 过期或失效,弹窗"登录已过期",点确定才清 token 跳登录页 (避免 race condition 把有效 token 误清)。

## 常见问题

### Q: 浏览器刷新后还是显示老页面?
A: 强制刷新 Ctrl+Shift+R (Mac: Cmd+Shift+R) 清浏览器缓存。

### Q: 操作后没反应?
A: 看浏览器 Network 标签,确认请求 status code + response body 的 errCode/errMsg。

### Q: Token 过期怎么办?
A: 浏览器弹窗后输入账号密码续期,或直接刷新重新登录。
