# I18N — 国际化与本地化

> **SSOT**: 本文件是 pxshe_app 国际化 (i18n) 和本地化 (l10n) 的唯一权威指南。
> 任何 l10n 改动前必读,违反直接拒绝合并。

---

## 1. 技术选型 (SSOT)

**单一来源 = `intl` gen-l10n (ARB)**。

| 候选 | 选择 | 原因 |
|---|---|---|
| ✅ `intl` + `flutter_localizations` + `gen-l10n` (ARB) | **是** | 官方推荐, type-safe, IDE 自动补全 |
| ❌ `easy_localization` | **否** | 跟 `intl` 重复 (违反 § 51 防堆砌), 业务代码用 `context.tr()` 但**从没**配齐资源/widget (bug f219a19 教训) |
| ❌ `i18n_extension` | **否** | 社区, 维护少 |

---

## 2. 资源位置

| 项 | 路径 | 备注 |
|---|---|---|
| ARB 文件 (SSOT) | `lib/l10n/arb/app_{zh,en,ar}.arb` | `app_en.arb` 是 template (`l10n.yaml` 配 `template-arb-file`) |
| 生成代码 (不要手改) | `lib/l10n/gen/app_localizations*.dart` | `flutter gen-l10n` 自动生成 |
| 配置 | `l10n.yaml` | 必填项: `arb-dir` / `template-arb-file` / `output-localization-file` / `output-dir` |
| 启用 | `pubspec.yaml` `flutter: generate: true` | ✅ 已开, `flutter pub get` 自动触发生成 |

### ARB 文件格式

```json
{
  "@@locale": "zh",
  "appTitle": "pxshe",
  "@appTitle": { "description": "应用名称,显示在 title bar" },
  "layoutPageHome": "首页"
}
```

**命名约定**: `intl` gen-l10n 用 **camelCase** (e.g. `layoutPageHome`, **不是** `layoutPage.home`)。
`easy_localization` 用 **dot notation** (`layoutPage.home`) — 那是历史包袱, 已删。

### 当前支持的语言

| Locale | 翻译 | 备注 |
|---|---|---|
| `en` (English) | ✅ 完整 | Template |
| `zh` (中文) | ✅ 完整 | 主要用户 |
| `ar` (العربية) | ⚠️ 用英文 placeholder | 待 L10n 阶段补翻译 |
| `es` (Español) | ⚠️ 在 Constants 列出但**没** ARB 文件 | 待补 (阶段 3 业务模块前) |

## 3. 业务代码怎么用

### 3.1 调翻译 (最常用)

```dart
import 'package:pxshe_app/_shared/l10n/gen/app_localizations.dart';

@override
Widget build(BuildContext context) {
  final l10n = AppLocalizations.of(context);  // ← 一次性取
  return Text(l10n.layoutPageHome);  // ← 直接用
}
```

**禁止**:
- ❌ `context.tr('layoutPage.home')` (easy_localization API, 已删)
- ❌ `AppLocalizations.of(context)!.layoutPageHome` 每次都调 (多一次 lookup, 浪费, 而且 nullable 检查不优雅 — 用 `final l10n = ...`)

### 3.2 切换语言 (用户在 settings 改)

```dart
context.read<LocaleCubit>().setLocale(Locale('zh'));
```

`LocaleCubit` (`lib/_shared/blocs/locale_cubit.dart`) 会:
1. emit 新 locale
2. `MaterialApp.router` 收到 `locale:` 变化 → 重 build → 整个 app 重新渲染
3. 持久化到 `THEME_BOX` (跟 `ThemeModeCubit` 复用同一 box), 下次启动恢复

### 3.3 读当前 locale (e.g. 显示当前语言)

```dart
final locale = Localizations.localeOf(context);
```

**不**用 `context.locale` (easy_localization API, 已删)。

### 3.4 加新 key 流程 (5 步)

1. **`app_en.arb`** 加:
   ```json
   "myNewKey": "My new English string",
   "@myNewKey": { "description": "说明这 key 干嘛" }
   ```
2. **`app_zh.arb`** 加 `"myNewKey": "我的新中文"`
3. (`app_ar.arb` / 其他 locale 同理, 没翻译用英文 placeholder)
4. **跑 `flutter gen-l10n`** — 自动更新 `lib/l10n/gen/`
5. **业务代码** 调 `AppLocalizations.of(context)!.myNewKey`

## 4. 启动配置 (硬约束 — 缺一抛 `LocalizationsNotFoundException`)

`lib/app/view/app.dart` `MaterialApp.router` **必须**配齐 3 件事:

```dart
MaterialApp.router(
  // ... theme / routerConfig / builder ...
  localizationsDelegates: const [
    AppLocalizations.delegate,
    GlobalMaterialLocalizations.delegate,
    GlobalCupertinoLocalizations.delegate,
    GlobalWidgetsLocalizations.delegate,
  ],
  supportedLocales: AppLocalizations.supportedLocales,
  locale: localeCubitState,  // ← 让 LocaleCubit 驱动
  builder: (innerContext, child) {
    AppModules.initAfterRunApp(innerContext);  // ← 见 ARCHITECTURE § 6.6
    return child!;
  },
)
```

**为啥要在 `builder` 里调 `initAfterRunApp`**: 那里 `innerContext` 经过了 MaterialApp + Localizations + Router, `AppLocalizations.of(context)` 不会 null。`App.build` 顶层调会 null 异常 (bug 3b1fca8)。

## 5. 测试 widget 怎么 mock l10n

`flutter_test` 测 widget 时, 需要包 `MaterialApp` 提供 `Localizations`:

```dart
testWidgets('HomePage renders app title', (tester) async {
  await tester.pumpWidget(
    MaterialApp(
      localizationsDelegates: AppLocalizations.localizationsDelegates,
      supportedLocales: AppLocalizations.supportedLocales,
      home: const HomePage(...),
    ),
  );
  expect(find.text('pxshe'), findsOneWidget);  // ← appTitle 来自 en.arb
});
```

**注**: 项目 253 个测试**目前没有**真渲染过需要 `AppLocalizations` 的 widget (LandingPage / SettingsPage / HomePage 导航 / LoginPage tooltip)。L10n 阶段 4 (集成测试) 要补这些 widget test。
